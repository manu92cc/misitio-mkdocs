¡Bienvenid@!

Esta página es un ejemplo de lo que se puede desarrollar con la herramienta MkDocs. Para una información más detallada puedes visitar mkdocs.org.

¿Qué es MkDocs?

MkDocs es una herramienta de documentación para crear sitios web estáticos. Es bastante sencillo y ligero de utilizar. Mediante el lenguaje markdown es posible realizar cualquier documentación necesaria para tus proyectos.

Cómo utilizar MkDocs

Para utilizar MkDocs sólo necesitaremos nuestro equipo con conexión a internet. En este caso utilizaremos Ubuntu (En windows los comandos pueden variar un poco, pero la teoría es la misma). Todo el proceso puede realizarse con el terminal.

Instalación

Para instalar MkDocs necesitaremos tener instalado python en nuestro equipo. Para comprobar qué versión de python tenemos instalada utilizaremos el comando:

pyton --version

En caso de que no tengamos python instalado solo habrá que instalarlo mediante el comando

sudo apt install python3

Instalamos Pip

Instalamos el gestor de paquetes de python Pip. (Éste paso solo es necesário una vez si no lo tienes ya instalado)

apt-get install python-pip

Instalamos MkDocs

Tan solo falta instalar el propio mkdocs mediante el comando:

pip install mkdocs

Y ya tendríamos todo listo para empezar a utilizarlo. Sencillo, ¿a que sí?

Comandos básicos

Para crear nuestros proyectos tendremos que conocer los siguientes comandos:

  • mkdocs new [directorio] - Crear un proyecto nuevo en directorio.
  • mkdocs serve - Ejecuta en el servidor indicado nuestro proyecto.
  • mkdocs serve -a [IP]:[PUERTO] - Si queremos utilizar una dirección ip en concreto.
  • mkdocs build - Construye el proyecto por nosotros.
  • mkdocs -h - Muestra la ayuda en el terminal.

Puedes obtener más información de todos los comandos en mkdocs.org.

Crear y lanzar primer proyecto

Vamos a crear una primera web de ejemplo. Antes de nada preparamos una nueva área de trabajo(opcional, pero recomendable). Puede ser la que quieras. En mi caso será una nueva carpeta en el Escritorio llamada MiSitio.

cd Escritorio
mkdir MiSitio
cd MiSitio

Ahora creamos nuestro proyecto, cuyo nombre será prueba.

mkdocs new prueba

Esto nos creará los elementos básicos para poder trabajar:

  • /prueba - La carpeta del proyecto que acabamos de crear.
    • /docs - Donde se almacenarán todos nuestros archivos .md
      • index.md - Cada archivo será una página diferente
    • mkdocs.yml - Éste será el archivo de configuración de nuestro proyecto

Para ver que todo funciona correctamente podemos lanzar nuestro sitio web con:

cd prueba
mkdocs serve

Y, si todo ha salido bien, nos dirá la dirección virtual de nuestro sitio web, en mi caso, http://127.0.0.1:8000/. Accedemos podremos ver algo como esto:

¡Listo! Ya tenemos nuestro sitio virtual abierto y funcionando. Ahora solo falta configurarlo para mostrar nuestra información.

Configurar nuestro sitio web

Configuración de página Inicio: index.md

Podemos modificar el archivo index.md a nuestro antojo para crear la página que queramos mediante lenguaje markdown. Por ejemplo:

# ¡Hola!

## Éste es mi sitio web

Espero que te guste. _;)_

Si guardamos el archivo veremos que la página se refresca automáticamente mostrando lo que acabamos de añadir al archivo index.md. También podemos crear más archivos .md en la carpeta docs para enlazarlos mediante el archivo mkdocs.yml. Veámos como.

Configuración de mkdocs.yml

Este archivo es el archivo maestro de nuestro sitio. Aqui podemos cambiar el título de la página, añadir más páginas, editar el tema y otras opciones.

Por ejemplo. Vamos a crear otro archivo en la carpeta docs/ con el nombre sobremi.md. Y en su interior escribiremos una descripción nuestra como ejemplo y guardaremos el archivo.

# Sobre mí

Soy una persona muy guapa.

Ahora, en el archivo de configuración mkdocs.yml añadiremos lo siguiente (y de paso cambiaremos el nombre de nuestro sitio por el que queramos, por ejemplo, Mi sitio web:

site_name: Mi sitio web

nav:        # Este elemento vinvulará nuestras páginas por nosotros.
    Inicio: index.md    # Ésta será nuestra página de inicio.
    Sobre mi: sobremi.md    # Ésta será nuestra página de presentación.

Una vez guardado veremos que en nuestra página aparece un nuevo enlace en la barra de navegación que nos dirige hacia el archivo

Fácil, ¿no?

Cambiar el tema de nuestrop sitio web

En éste mismo archivo podemos cambiar el tema de nuestras páginas. En mi caso voy a cambiarlo por el tema readthedocs. Para ello añadimos la siguiente línea en mkdocs.yml

# Aquí estará el código anterior
theme: readthedocs

Una vez guardado vemos como cambia el tema por el que hemos elegido. Puedes buscar mas temas aquí.

Construir el proyecto

Una vez creas que ya has terminado tu proyecto solo tienes que construirlo mediante el comando:

mkdocs build

Esto te creará todas las carpetas y archivos necesários para poder dar por terminado tu proyecto.

Y ¡Todo listo!

Ahora ya tienes tu proyecto terminado y puedes poblucar tu sitio web. Te recordamos que esto ha sido un pequeño tutorial, si quieres más información puedes consultar las páginas oficiales de MkDocs, donde podrás encontrar información más detallada y completa de todos estos pasos y más opciones que ofrece la herramienta.
